## Descripción del Curso

Este curso proporciona las herramientas y metodologías necesarias para realizar un handoff efectivo desde Figma hacia el equipo de desarrollo. Aprenderás a preparar, documentar y comunicar tus diseños de manera que los desarrolladores puedan implementarlos con precisión, reduciendo fricciones y reprocesos en el flujo de trabajo.

---

## Objetivos de Aprendizaje

### Objetivo General
Dominar el proceso de handoff de diseño a código utilizando Figma, estableciendo una comunicación clara y eficiente con el equipo de desarrollo.

### Objetivos Específicos
Al finalizar el curso, los participantes serán capaces de:

1. **Compartir diseños** de forma accesible sin requerir que los desarrolladores creen cuentas en Figma
2. **Preparar archivos** con la estructura y nomenclatura adecuada para facilitar la implementación
3. **Documentar estados** e interacciones de componentes de manera clara
4. **Utilizar Dev Mode** para extraer especificaciones técnicas precisas
5. **Establecer flujos de feedback** efectivos entre diseño y desarrollo
6. **Exportar assets** optimizados para diferentes plataformas

---

## Syllabus

### Módulo 1: Fundamentos del Handoff (15 min)
- ¿Qué es el handoff y por qué es importante?
- Problemas comunes en la comunicación diseño-desarrollo
- El rol del diseñador en el proceso de implementación

### Módulo 2: Compartir Diseños sin Fricción (15 min)
- Configuración de permisos y acceso público/privado
- Links de solo lectura para stakeholders
- Modo de visualización para desarrolladores sin cuenta
- Embed de prototipos en documentación

### Módulo 3: Preparación del Archivo para Handoff (20 min)
- **Organización de capas y páginas**
  - Nomenclatura consistente (BEM, atomic design)
  - Estructura de frames y grupos
- **Auto Layout y Constraints**
  - Cómo configurarlos para que reflejen el comportamiento responsive
- **Design Tokens y Variables**
  - Colores, tipografía, espaciados
  - Integración con sistemas de diseño
- **Componentes y Variantes**
  - Documentación de props y estados

### Módulo 4: Estados, Animaciones y Prototipos (15 min)
- Documentación de estados de componentes (hover, active, disabled, error, etc.)
- Especificación de micro-interacciones y transiciones
- Prototipos interactivos: cuándo usarlos y cuándo no
- Anotaciones de comportamiento y lógica condicional

### Módulo 5: Dev Mode y Especificaciones Técnicas (15 min)
- Introducción a Dev Mode de Figma
- Extracción de código CSS/Swift/Android
- Medidas, espaciados y tipografía
- Exportación de assets (SVG, PNG, WebP)
- Plugins útiles para el handoff

### Módulo 6: Feedback y Colaboración Continua (10 min)
- Uso de comentarios en Figma
- Flujos de revisión y aprobación
- Gestión de cambios y versionado
- Herramientas complementarias (Loom, Slack, Notion)
- Ceremonias de sincronización diseño-desarrollo

## Requerimientos

### Para los Participantes

**Conocimientos previos:**
- Manejo básico de Figma (creación de frames, componentes básicos)
- Comprensión general del flujo de diseño de producto
- Familiaridad con conceptos básicos de desarrollo web (deseable, no obligatorio)

**Técnicos:**
- Computadora con navegador actualizado (Chrome, Firefox, Safari, Edge)
- Cuenta de Figma (versión gratuita es suficiente)
- Conexión a internet estable

### Para el Instructor

**Materiales a preparar:**
- Archivo de Figma de ejemplo con componentes listos para handoff
- Archivo de Figma con "errores comunes" para demostración
- Checklist de handoff imprimible/descargable
- Slides de apoyo

**Herramientas:**
- Figma con plan profesional (para demostrar Dev Mode completo)
- Herramienta de grabación de pantalla (si es virtual)
